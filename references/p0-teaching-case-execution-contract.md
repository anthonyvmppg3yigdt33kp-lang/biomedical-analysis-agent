# P0 teaching-case execution evidence contract

`references/p0-teaching-cases.json` remains a public, candidate-only design registry. A successful local run never mutates its `not-executed` or candidate maturity fields.

Verified executions are recorded separately in the private-only
`assets/private-corpus-index/p0-teaching-case-executions.json`. The schema uses a
domain-discriminated `oneOf`: a record must satisfy exactly one complete domain
contract. Domain fields cannot be mixed or silently borrowed from another branch.

The `visualization` branch remains backward compatible. It binds one read-only
input, task-local `python-uv` evidence, fresh and cache-restored runs, exactly nine
hash-identical machine artifacts, two native-reviewed figures, and one canonical
`DELIVERED` formal run with its plan, environment, ledger, reports, figures,
promotion evidence, and read-only resume audit.

The `single-cell` branch is a separate Python reduced-real-data fixture contract.
It requires:

- explicit task-local `run` authorization, read-only sources, no global changes,
  and no remote publication;
- the complete public-integrity locator set and metadata sidecar binding, while
  keeping private paths and hashes out of public schema fixtures and tests;
- a frozen task-local relocated Conda/Python snapshot whose source and target tree
  hashes agree, with no active external/editable locator, external runtime
  reference, or generated bytecode in the frozen target;
- zero fresh and cached-resume return codes, cache revalidation of stages
  `00-intake`, `01-environment`, and `02-analysis`, and an exact 31-artifact index;
- checkpoints `00-intake` through `04-delivery-report`, with payload trees rebuilt
  from live bytes and the latest valid checkpoint fixed at `04-delivery-report`;
- six hash/size/dimension-bound figures opened at native resolution, a terminal
  `PASS_WITH_MINOR_FINDINGS` review, and a delivery report bound to that review;
- donor-preserving summaries, explicit all-donors-or-none doublet handling,
  process-local thread limits, memory-safety evidence, and no inferential test.

For single-cell, `data-verified` applies only to the deterministic reduced
real-data teaching fixture. It does not verify full-data execution, population
prevalence, case-control effects, causal or clinical claims, calibrated doublet
prevalence, or reference-validated cell identities. A record cannot obtain this
maturity from schema shape alone: live input, environment, checkpoint, artifact,
native-review, delivery, QA, and claim-boundary bindings must all pass.

The `spatial-transcriptomics` branch is an independent real-data descriptive
contract for one 10x Visium section. It requires:

- all four public/private Spotiphy teaching-input locators, byte sizes and SHA-256
  values to agree without redistributing the H5AD, histology or ZIP bytes;
- explicit spot, section and animal/specimen units, with exactly one independent
  inference unit and population inference disabled;
- a frozen task-owned `uv` environment and managed CPython runtime, with the
  install-time runtime-tree contract, lock files, package versions and executable
  hashes bound to the private evidence record;
- the ordered ten-stage path from `S00_INTAKE` through
  `S95_VISUALIZE_INTERPRET`, whose checkpoint payload trees are rebuilt from live
  bytes;
- an exact 102-artifact index and matching 102-line append-only ledger, four
  original/final image pairs opened at native resolution, and a terminal
  `PASS_WITH_MINOR_FINDINGS` review;
- quantitative coordinate/image agreement, count-matrix integrity, graph and QC
  sensitivity evidence, and a claim ceiling restricted to one-section
  descriptive patterns;
- an explicit `blocked` Spotiphy model branch when its separate compatibility
  environment is unavailable. No KMeans, Moran ranking or other method may be
  recorded as a deconvolution substitute.

For spatial data, `data-verified` covers the executed coordinate-faithful
descriptive core only. It does not verify Spotiphy deconvolution, validated cell
types or anatomical domains, permutation/FDR-controlled spatial discoveries,
replication across animals, population effects, mechanism or causality. Rejected
visual runs and recovered cross-task environment contamination remain registered
as failure evidence rather than being silently discarded.

The live validator re-hashes every declared private source and evidence file,
rebuilds checkpoint and output-tree contracts, verifies the artifact index as an
exact set, cross-binds native-reviewed pixels to indexed figures, reconciles the
delivery checkpoint and final Figure Notes, and confirms that the public candidate
registry is still immutable. Duplicate execution IDs or case executions, missing
bindings, partial output, non-zero returns, global environment changes, false
cache claims, or scientifically over-broad delivery invalidate the private
execution record; they never promote or rewrite the public candidate.

Use `scripts/register_p0_teaching_case_execution.py` to append one private
candidate atomically. It refuses duplicate execution IDs and duplicate case
executions, checks the public case/domain pairing, validates the complete union
schema, and only then replaces the registry file.

Validate with:

```powershell
python scripts/validate_p0_teaching_case_executions.py --verify-live
```

## Explain-only methodology audits

An `explain` teaching case is not recorded as a data execution. Its private
overlay lives in `assets/private-corpus-index/p0-methodology-audits.json` and is
validated against `references/schemas/p0-methodology-audit.schema.json`.

The literature-methodology contract requires read-only source access, zero source
code execution, zero package installation, no global changes, and no source-byte
redistribution. It re-hashes the private article, ordered source code and native
reference figure; rebuilds every checkpoint payload tree; reconciles the exact
formal artifact index and ledger; checks the MethodCard, PackageCard, FigureCard,
risk ledger and claim ceiling; and verifies the user delivery independently.

For the ROGUE negative-control case, `native-reviewed` means that the original
plot was opened at native pixels and its style/code lineage was understood. It
does not promote the source code to executable maturity and does not validate a
biological ROGUE result. The randomized A/B/C/D cell groups remain a blocking
pseudoreplication example. Register and validate with:

```powershell
python scripts/register_p0_methodology_audit.py <private-audit-candidate.json>
python scripts/validate_p0_methodology_audits.py --verify-live
```
